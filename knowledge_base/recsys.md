## 推荐系统解决的问题

**核心目标**：在信息过载的场景下，从海量物品（item）中为用户（user）挑选出其可能感兴趣的内容，本质是一个**匹配问题**即学习 user 和 item 之间的匹配函数：\

score=f(user,item,context)\

**与搜索的区别**：搜索是"人找货"（有明确 query），推荐是"货找人"（无显式 query，依赖用户画像和历史行为）。\
## 核心技术范式

### 1 协同过滤（Collaborative Filtering, CF）

基于"物以类聚，人以群分"的假设，不依赖内容特征，只用用户-物品的交互矩阵。

- **UserCF**：找相似用户，推荐相似用户喜欢的东西\
- **ItemCF**：找相似物品，推荐用户历史喜欢物品的相似物品\
- **矩阵分解（Matrix Factorization, MF）**：将用户-物品交互矩阵R 分解为两个低维矩阵 R≈U⋅V<sup>T</su>，U 是用户隐向量，V 是物品隐向量，点积即为预测分数\

### 2 基于内容的推荐（Content-based）

利用物品自身的属性特征（文本、类别、标签等）和用户历史偏好做匹配，不依赖其他用户的行为，可以缓解冷启动问题。

### 3 双塔模型（Two-Tower Model）

召回阶段的主流范式：user 塔和 item 塔分别编码成低维向量，通过内积或余弦相似度衡量匹配程度。

- 优点：user 向量和 item 向量可以离线预计算，item 向量建索引（如 Faiss），线上只需通过 ANN（近似最近邻）检索，速度快\
- 缺点：user 和 item 特征在塔内独立编码，**双塔之间没有交叉**，表达能力弱于精排模型\
### 4 深度学习精排模型

- **Wide & Deep**：Wide 部分（线性模型）负责记忆（memorization），Deep 部分（DNN）负责泛化（generalization）\
- **DeepFM**：用 FM（因子分解机）自动学习特征二阶交叉，替代 Wide&Deep 中人工设计的交叉特征\
- **DIN（Deep Interest Network）**：引入 attention 机制，让用户历史行为序列针对当前候选物品做加权聚合（而不是简单 pooling）\
- **DIEN**：在 DIN 基础上建模用户兴趣的演化过程（序列/时序建模）\
## 常见评价指标

| 指标                                | 含义                       | 适用阶段        |
| --------------------------------- | ------------------------ | ----------- |
| **Precision@K / Recall@K**        | Top-K 结果中的准确率/召回率        | 召回、排序       |
| **AUC**                           | 衡量正负样本排序能力（不受阈值影响）       | 精排（CTR预估）   |
| **NDCG（归一化折损累积增益）**               | 考虑排序位置的相关性加权指标，位置越靠前权重越高 | 排序质量评估      |
| **CTR / CVR**                     | 点击率 / 转化率                | 线上效果、离线预估目标 |
| **多样性（Diversity）/ 覆盖率（Coverage）** | 推荐结果的丰富程度、长尾物品曝光情况       | 重排、系统健康度    |
## 冷启动问题（Cold Start）

- **用户冷启动**：新用户无历史行为 → 依赖注册信息、内容特征、热门兜底、探索性推荐（exploration）\
- **物品冷启动**：新物品无交互数据 → 依赖内容特征做基于内容的匹配，或加大新物品的曝光探索权重\
- **系统冷启动**：整个平台刚上线，缺乏数据积累 → 依赖规则、专家知识、迁移学习\
